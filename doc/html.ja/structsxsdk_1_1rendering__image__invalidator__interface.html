オーバライド可。レンダリングスレッド、またはShadeGridサーバに割り当てられたレンダリング領域が通知される。

レイトレース処理が行われる前に呼ばれるので、これから計算を開始する場所を表示するときに使用する。同じ領域に対して複数のスレッドが割り当てられる場合がある。

複数のスレッドから同時に呼ばれるため、必要に応じて排他制御をクライアントコード側で実装する。

引数:
area レンダリングスレッドに割り当てられた領域
thread_index スレッドの番号。server がtrueの場合はShadeGridの番号
server ShadeGridサーバに割り当てられた場合はtrueが渡される

int  thread_index,
bool  server,
void *  aux = 0

オーバライド可。レイトレース処理の完了した領域が通知される。

rendering_interface::get_image_interface, rendering_interface::get_image_layer_interface で取得したイメージのうち、area で渡された領域が更新箇所となる。

レンダリング完了まで処理を続けた場合、rendering_interface::total_number_of_rendering_blocks で取得できる数と同じ回数が呼び出される。

複数のスレッドから同時に呼ばれるため、必要に応じて排他制御をクライアントコード側で実装する。

引数:
area レンダリングの完了した領域
thread_index スレッドの番号。server がtrueの場合はShadeGridの番号
server ShadeGridサーバに割り当てられた場合はtrueが渡される

virtual void sxsdk::rendering_image_invalidator_interface::rendering_finished ( void *  aux = 0  )  [inline, virtual]

オーバライド可。レンダリングが終了したときに呼ばれる。

この関数の呼出し前に取得したレンダリングイメージは破棄されるため、rendering_interface::get_image_interface, rendering_interface::get_image_layer_interface で再取得する必要がある。

オーバライド可。レンダリングが開始されたときに呼ばれる。

レンダリングイメージが生成された後に呼ばれる。この関数の呼出し前に取得したレンダリングイメージは破棄されるため、rendering_interface::get_image_interface, rendering_interface::get_image_layer_interface で再取得する必要がある。
